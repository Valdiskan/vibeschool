ssssd# VibeSchool — платформа вайбкодинга для школьников 5–9 классов

Ученик описывает программу по-русски → YandexGPT пишет код (Python/JS) → код выполняется в изолированной песочнице Piston → учитель видит промпты класса в дашборде.

**Стек:** Next.js 15 (App Router) · TailwindCSS · Supabase (Auth + PostgreSQL) · YandexGPT API · Piston API · CodeMirror.

**Бюджет:** 0 ₽. Vercel Hobby + Supabase Free + публичный Piston бесплатны; YandexGPT — копейки за yandexgpt-lite (и есть демо-режим вообще без ключей).

---

## Запуск за 15 минут

### 1. Supabase (база + авторизация)

1. [supabase.com](https://supabase.com) → New project (Free tier, регион любой).
2. **SQL Editor → New query** → вставь содержимое `supabase/schema.sql` → **Run**.
3. **Authentication → Sign In / Up → Email** → выключи **Confirm email** (для пилота, чтобы дети не возились с почтой).
4. **Project Settings → API** → скопируй `Project URL` и `anon public` ключ.

### 2. Локальный запуск

```bash
npm install
cp .env.example .env.local
# впиши NEXT_PUBLIC_SUPABASE_URL и NEXT_PUBLIC_SUPABASE_ANON_KEY
npm run dev
```

Открой http://localhost:3000 — приложение уже полностью рабочее:
генерация кода идёт в **демо-режиме** (шаблоны по ключевым словам), запуск кода — через настоящий Piston.

### 3. YandexGPT (настоящая генерация)

1. [console.yandex.cloud](https://console.yandex.cloud) → создай платёжный аккаунт и каталог (folder).
2. Создай сервисный аккаунт с ролью `ai.languageModels.user`, для него — **API-ключ**.
3. В `.env.local`:

```
YANDEX_API_KEY=AQVN...
YANDEX_FOLDER_ID=b1g...
YANDEX_MODEL=yandexgpt-lite
```

`yandexgpt-lite` — самая дешёвая модель, для учебных консольных программ её достаточно. Пилот на 10 учеников × 8 занятий обойдётся в десятки рублей.

### 4. Деплой на Vercel (демо-ссылка)

1. Залей репозиторий на GitHub.
2. [vercel.com](https://vercel.com) → Add New Project → импортируй репозиторий.
3. В **Environment Variables** добавь все переменные из `.env.local`.
4. Deploy → получишь ссылку вида `vibeschool.vercel.app` для пилота.

## Как пользоваться

**Учитель:** регистрируется с ролью «Учитель» и придумывает **код класса** (например `7B-2026`). Раздаёт код ученикам. В разделе **Класс** видит список учеников и их последние промпты.

**Ученик:** регистрируется с кодом класса → открывает **Уроки** (4 встроенных урока, Markdown) → из урока попадает в **Студию** с уже подставленным стартовым промптом → генерирует, запускает, улучшает.

**Ввод данных:** песочница не интерактивна, поэтому в Студии есть поле «что программа введёт с клавиатуры» — ответы пишутся заранее, по строке на каждый `input()`.

## Безопасность (реализовано)

| Требование ТЗ | Где |
|---|---|
| Экранирование промптов | `src/lib/sanitize.ts`: NFC-нормализация, вырезание управляющих/невидимых символов, лимит 800 символов; промпт оборачивается в маркеры `<<<НАЧАЛО/КОНЕЦ ЗАДАНИЯ>>>`, системный промпт велит игнорировать инструкции внутри |
| Фильтр лексики | `src/lib/profanity.ts`: нормализация латиницы-«двойников» и leetspeak, поиск по корням |
| Песочница | `/api/run`: Piston с `run_timeout: 5000` (5 с), `run_memory_limit: 128 МБ`; сеть и файлы внутри Piston недоступны |
| Доступ к API | оба роута требуют авторизации Supabase |
| Данные | RLS: ученик видит только своё; учитель — только свой класс (`supabase/schema.sql`) |
| Детский контент | системный промпт YandexGPT: при неподходящем задании модель отвечает `REFUSED`, ученик получает мягкий отказ |

## Структура проекта

```
src/
  app/
    page.tsx                лендинг
    login/  register/       авторизация (email+пароль, роль, код класса)
    studio/page.tsx         главный экран: промпт → код → запуск
    lessons/                4 встроенных урока (Markdown)
    teacher/page.tsx        дашборд учителя
    api/generate/route.ts   YandexGPT (+ демо-режим без ключей)
    api/run/route.ts        прокси к Piston с лимитами
  lib/
    lessons.ts              контент уроков
    profanity.ts            фильтр лексики
    sanitize.ts             санитизация промпта, извлечение кода из ответа
    demoGenerate.ts         офлайн-шаблоны для демо-режима
    supabase/               клиенты (browser / server)
  components/               UI-кит, редактор CodeMirror, навбар
middleware.ts               защита /studio и /teacher
supabase/schema.sql         таблицы, триггер регистрации, RLS
```
